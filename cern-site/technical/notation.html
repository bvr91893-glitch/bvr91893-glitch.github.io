Notation

A. BNF notation for syntax

This section has three parts:

  1. a straight copy of a section of RFC #822 Standard for ARPA Internet Text Messages,
  2. changes and additions,
  3. a set of rules that are used everywhere and listed here once.

(a) Notational Conventions

1. Rule Naming

Angle brackets "<" and ">" are not used, in general. The name of a rule is simply the name itself.

2. RULE1 | RULE2: Alternatives

Elements separated by a vertical bar are alternatives.

3. (RULE1 RULE2): Local Alternatives

Elements enclosed in parentheses are treated as a single element.

4. *RULE: Repetition

The character * preceding an element indicates repetition.

5. [RULE]: Optional

Square brackets enclose optional elements.

6. NRULE: Specific Repetition

This means exactly n occurrences of an element.

7. #RULE: Lists

A construct using # indicates a list of elements separated by commas.

8. ; Comments

A semicolon starts a comment to the end of the line.

(b) Changes and additions

  1. Nonterminals are written starting with a capital.
  2. A rule is written with the ::= notation.
  3. Alternatives are separated by a vertical bar rather than a slash.
  4. Some layout character sequences use C string conventions.

(c) General rules

HTTP data are written in line format, i.e., line breaks are significant.

Line breaks

CrLf ::= \r\n

Strings

A string is surrounded by double-quote characters.
